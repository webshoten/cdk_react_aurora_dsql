# 11-2. data-02

## 検証対象

- 医療者Web表示（React + GraphQL Lambda + S3 Presigned URL）

## 目的

- 画像ファイルを S3 に保存し、DB の画像パスと合わせて管理できることを確認する
- 画面操作からアップロードと一覧表示（サムネイル表示）までを確認する

## 設計（確定）

- S3 は `StorageStack` で管理する
- 11-2 で利用するオブジェクトキーは `image/` プレフィックス配下に限定する
- アップロードは GraphQL 経由で Presigned URL（PUT）を取得して実行する
- 登録は「S3 ファイル保存」と「DB への画像パス登録」をセットで行う
- 一覧取得は DB を正とし、表示時に必要な Presigned URL（GET）を返す
- 画面はテーブル表示とし、サムネイル列で小サイズ画像を確認できるようにする

## データフロー（11-2）

1. 画面で画像を選択する
2. `createImageUploadUrl` で `image/...` 向けアップロード URL を取得する
3. ブラウザから S3 に PUT する
4. `registerImage` で DB に画像パスを登録する
5. `images` Query で一覧再取得し、サムネイルを表示する

## 運用方針

- 11-2 の初期スコープでは `image/` 以外の用途は扱わない
- 将来の Firehose / 顔ぼかし連携は `StorageStack` 内で別プレフィックスまたは別バケットとして拡張する

## 決定ログ

- 2026-04-27: S3 管理を `StorageStack` へ分離する方針を採用
- 2026-04-27: 11-2 は `image/` プレフィックスのみを対象にする方針を採用
