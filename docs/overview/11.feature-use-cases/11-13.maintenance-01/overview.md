# 11-13. maintenance-01

## 検証対象

- メンテナンス画面の表示制御
- バイパス cookie による通常画面への通過制御

## 前提（確定）

- React 単体では Next.js middleware 相当の制御はできない前提で設計する
- エッジでの振り分けを前提とし、`CloudFront + Lambda@Edge` を候補に含める

## 実装スコープ（11-13）

- メンテナンスモード ON/OFF 時の画面切り替え方式を定義する
- バイパス cookie の付与条件と有効期限の方針を定義する
- 通常画面へのアクセス許可/拒否の判定仕様を定義する
- フロント側の表示（メンテナンス画面）と API 呼び出し時の挙動を整理する

## スコープ外（11-13ではやらない）

- 本番切り替え運用の自動化
- 監視/通知の最終設計
